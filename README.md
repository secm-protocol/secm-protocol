# SECM Protocol

**System of Multilayer Knowledge Engineering** — a Knowledge Engineering Protocol that
transforms heterogeneous representations of reality into structured metadata, allowing
independent transformation engines to converge toward **explainable directional
hypotheses** that reduce uncertainty in human decision making.

The protocol never determines destiny. It estimates directions.

> The purpose of SECM is not to tell people who they are. It is to help people,
> organizations and societies navigate reality with greater clarity, lower uncertainty
> and better alignment between potential, context and direction.
> — RFC-0000 §24

## What SECM is not

Not a numerology system. Not astrology. Not a personality test. Not an AI assistant.
Not a prediction engine. Not a replacement for human judgment. (RFC-0000 §1, §20)

SECM decomposes disciplines into computational transformations — and every
transformation earns its weight through **validated evidence** (RFC-0001 §6). Nothing
is adopted by faith. Nothing is hidden from audit.

## Status

**Pre-v1.0 — the vertical slice is complete.** The constitution is ratified and
**18 RFCs are active**. The full RX pipeline runs end-to-end on `develop` with a
green test suite: 10-question intake → FE-005 Context + FE-006 Behavioral → CKE
convergence → human-readable reading → outcome registration → Brier calibration.
Every artifact validates against the canonical schema, every sentence in a reading
cites its evidence, and every confidence score explains itself. Next: real-world
usage, ingestion pipelines, and the road to the v1.0 release —
*no line of code without a corresponding RFC.*

## Architecture (constitutional layers)

```
Constitution → Protocol Core → Foundation Engines → Extension Engines → Solvers
```

Nine Foundation Engines define the protocol's identity (RFC-0000 §8). The human being
is permanently at the center (§5). Outputs are always explainable, weighted, versioned,
traceable and confidence-scored — never deterministic (§13).
